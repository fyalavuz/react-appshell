## Requirements

`@appshell/react` is built with **Tailwind CSS v4**. Components use Tailwind utility classes and reference CSS custom properties for colors.

## Header themes

The `Header` component supports three built-in themes via the `theme` prop:

| Theme | Background | Text | Borders |
| --- | --- | --- | --- |
| `light` | `bg-white` | `text-gray-900` | `border-gray-200` |
| `primary` | `bg-primary` | `text-primary-foreground` | `border-primary/80` |
| `dark` | `bg-gray-900` | `text-white` | `border-gray-800` |

```tsx
<Header theme="dark" behavior="fixed" logo={<span>App</span>} />
```

The `primary` theme references the `--color-primary` and `--color-primary-foreground` CSS custom properties. Define these in your global CSS:

```css
:root {
  --color-primary: oklch(0.55 0.2 250);
  --color-primary-foreground: oklch(1 0 0);
}
```

## Footer and Content styling

The `Footer` tab-bar and mini variants use `bg-background/95` and `border-t` classes. The `Content` component applies `flex-1` only. Customize them with the `className` prop:

```tsx
<Footer
  variant="tab-bar"
  className="bg-gray-950/95 border-gray-800"
>
  {/* items */}
</Footer>
```

## CSS custom properties

The components reference the following Tailwind CSS v4 theme variables. Override them in your CSS to change the appearance globally:

| Variable | Used by | Purpose |
| --- | --- | --- |
| `--color-primary` | Header (primary theme), FooterItem active state | Primary brand color |
| `--color-primary-foreground` | Header (primary theme) | Text on primary backgrounds |
| `--color-background` | Footer (tab-bar, mini) | Page background |
| `--color-foreground` | FooterItem | Default text color |
| `--color-muted-foreground` | FooterItem (inactive) | Dimmed text |
| `--color-destructive` | FooterItem badge | Badge background |
| `--color-border` | Footer | Border color |
| `--color-accent` | Header mobile menu hover | Hover backgrounds |

## className overrides

Every component accepts a `className` prop. Classes are merged with `clsx` and `tailwind-merge`, so your overrides take precedence:

```tsx
<Header className="bg-blue-600 text-white" theme="light" />
```

## Dark mode

Wrap your layout in a container with the `dark` class to switch Tailwind's dark variant:

```tsx
<div className="dark">
  <AppShell>
    <Header theme="dark" />
    <Content className="bg-gray-900 text-white">
      {/* content */}
    </Content>
    <Footer className="bg-gray-950 border-gray-800">
      {/* items */}
    </Footer>
  </AppShell>
</div>
```

The built-in `dark` header theme pairs well with this approach, as seen in the Music Player and Dashboard examples.
