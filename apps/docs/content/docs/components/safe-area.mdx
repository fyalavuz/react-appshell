---
title: SafeArea
description: Automatic safe area inset padding for notched and rounded-corner devices.
---

The `SafeArea` component applies `env(safe-area-inset-*)` CSS padding to protect content from device notches, home indicators, and rounded corners.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `edges` | `SafeAreaEdge[]` | `["top", "bottom", "left", "right"]` | Which edges to pad. Values: `"top"`, `"bottom"`, `"left"`, `"right"`. |
| `className` | `string` | — | Additional CSS classes |
| `children` | `ReactNode` | — | Content to wrap |

## Usage

Apply safe area padding on all four edges:

```tsx
import { SafeArea } from "@appshell/react";

<SafeArea>
  <div>Content is padded away from device edges</div>
</SafeArea>
```

### Specific edges

Pad only the top and bottom:

```tsx
<SafeArea edges={["top", "bottom"]}>
  <div>Horizontally full-bleed, vertically safe</div>
</SafeArea>
```

### Bottom only

Useful for fixed footers:

```tsx
<SafeArea edges={["bottom"]}>
  <footer>This footer respects the home indicator</footer>
</SafeArea>
```

## How it works

`SafeArea` applies inline styles using CSS environment variables:

```css
padding-top: env(safe-area-inset-top, 0px);
padding-bottom: env(safe-area-inset-bottom, 0px);
padding-left: env(safe-area-inset-left, 0px);
padding-right: env(safe-area-inset-right, 0px);
```

Only the edges listed in the `edges` prop are applied. The fallback value is `0px`, so on devices without notches there is no extra padding.

## Viewport meta tag

For `env(safe-area-inset-*)` to return non-zero values, your HTML must include the `viewport-fit=cover` meta tag:

```html
<meta
  name="viewport"
  content="width=device-width, initial-scale=1, viewport-fit=cover"
/>
```

Without `viewport-fit=cover`, the browser treats the safe area insets as zero on all edges.

## Using with AppShell

The `AppShell` component has a `safeArea` prop that wraps children in `SafeArea edges={["top", "bottom"]}` automatically:

```tsx
<AppShell safeArea>
  {/* Header, Content, Footer */}
</AppShell>
```
