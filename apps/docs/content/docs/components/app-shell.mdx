---
title: AppShell
description: The root wrapper component that provides shared context and layout structure for your mobile app shell.
---

`AppShell` is the outermost wrapper for your app. It creates a full-height flex column layout and automatically includes `AppShellProvider` so all child components can share scroll and visibility state.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `safeArea` | `boolean` | `false` | Wraps children in a `SafeArea` with top and bottom edges |
| `className` | `string` | — | Additional CSS classes for the container |
| `children` | `ReactNode` | — | Header, Content, Footer, and other elements |

## Usage

```tsx
import { AppShell, Header, Content, Footer } from "@appshell/react";

export default function App() {
  return (
    <AppShell safeArea>
      <Header behavior="fixed" logo={<span>Logo</span>} />
      <Content>
        <p>Page content</p>
      </Content>
      <Footer variant="tab-bar">{/* FooterItems */}</Footer>
    </AppShell>
  );
}
```

## How it works

`AppShell` renders:

1. An `AppShellProvider` that holds shared state (header/footer visibility, scroll direction).
2. A `div` with `min-h-dvh` and `flex flex-col` to fill the viewport.
3. Optionally, a `SafeArea` wrapper with `edges={["top", "bottom"]}` when `safeArea` is `true`.

## Using AppShellProvider directly

If you need the context without the layout wrapper, use `AppShellProvider` on its own:

```tsx
import { AppShellProvider, useAppShell } from "@appshell/react";

function MyApp() {
  return (
    <AppShellProvider>
      <MyCustomLayout />
    </AppShellProvider>
  );
}
```
